# P4-A 事前审计报告：数据与交易流程现状分析

## 审计范围
对 NexusTrade 模拟交易平台在 P4-A 实施前的数据模型、交易流程、UI 状态管理进行全面审计。

---

## 一、Mock 数据现状（P4-A 前）

### 1.1 板块数据
- 热度排名（hotRank）为硬编码固定值，无计算逻辑
- 资金流向（capitalInflow）与板块涨跌幅无关联
- 无风险等级（riskLevel）字段
- 无趋势方向（trend）字段
- 板块间涨跌逻辑不一致：强势板块个股涨跌比例不合理

### 1.2 个股数据
- 涨跌幅与所属板块强度无关联
- 市盈率（PE）未区分正负（亏损公司无标注）
- 无风险提示（riskWarning）字段
- 开盘价、最高价、最低价与当前价无逻辑关系
- 成交量/成交额为独立数值，不反映板块热度

### 1.3 K 线数据
- 趋势线随机生成，与涨跌幅方向不一致
- 无 MA 均线指标
- 无分析文字说明
- 蜡烛图数据无 OHLC 逻辑一致性

### 1.4 订单数据
- 订单状态仅 4 种：submitted / filled / cancelled / partial_filled
- 无佣金（commission）计算
- 无印花税（stampTax）计算
- 订单 ID 格式简单（非真实委托编号格式）
- 时间戳为当前时间，非真实交易时段
- 无成交均价（avgFillPrice）
- 无部分成交数量（filledQuantity）
- 无拒绝原因（rejectReason）

---

## 二、交易流程现状（P4-A 前）

### 2.1 买入流程
1. 用户在个股页面看到买入按钮
2. 输入数量 → 直接提交
3. 无确认对话框
4. 无费用预估（佣金、税费）
5. 无风险提示

### 2.2 卖出流程
- 与买入流程完全对称，无差异处理
- 无持仓校验

### 2.3 订单页面
- 简单列表展示
- 无筛选/状态分类
- 无空状态设计
- 无加载状态

---

## 三、UI 状态组件现状（P4-A 前）

### 3.1 缺失的通用组件
- ❌ LoadingState — 加载骨架屏
- ❌ EmptyState — 空数据占位图
- ❌ ErrorState — 错误状态展示
- ❌ WarningBanner — 风险/警告横幅

### 3.2 现有状态处理
- 页面直接渲染，无 loading 过渡
- 数据为空时显示空白区域
- 无错误边界处理

---

## 四、问题总结

| 问题类别 | 具体问题 | 严重程度 |
|---------|---------|---------|
| 数据逻辑 | hotRank 无计算公式 | 中 |
| 数据逻辑 | 个股与板块涨跌不关联 | 高 |
| 数据逻辑 | K 线趋势与涨跌幅矛盾 | 中 |
| 交易流程 | 无确认对话框 | 高 |
| 交易流程 | 无佣金/税费计算 | 高 |
| 交易流程 | 无风险提示 | 高 |
| UI 状态 | 无 Loading/Empty/Error 组件 | 中 |
| 图表分析 | 无 MA 均线指标 | 中 |
| 图表分析 | 无分析文字说明 | 低 |

---

**审计结论**：数据模型缺乏真实市场的逻辑关系，交易流程缺少关键的安全确认环节，UI 缺少必要的状态组件。P4-A 需全面升级这三个维度。
