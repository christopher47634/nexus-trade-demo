# P4-A Phase 1：数据真实性改造方案

## 改造目标
将静态 Mock 数据升级为具有逻辑关系的"真实感"数据，让用户体验到类真实交易环境。

score = changePercent × 2 + (turnover_亿 ÷ 50)

- 正 score → capitalInflow 为正（资金净流入）

- 绝对值与 score 成正比

## 二、个股-板块关联

### 关联规则
1. **强势板块**（trend: up）：5只个股中4正1微负（获利回吐）
2. **弱势板块**（trend: down）：5只个股中4负1微正（避险/反弹）
3. 个股涨跌幅在板块涨跌幅 ±3% 范围内浮动

### 实例：光通信板块（+3.85%）
| 个股 | 涨跌幅 | 说明 |
|------|--------|------|
| 中际旭创 | +5.23% | 龙头股，领涨 |
| 新易盛 | +3.67% | 跟涨 |
| 天孚通信 | +2.45% | 跟涨 |
| 博创科技 | +6.12% | 高换手，活跃 |
| 亨通光电 | -0.28% | 获利回吐 |

### 市盈率合理性
- 正常公司：10-60倍
- 亏损公司（寒武纪、亿航智能）：PE 为负值
- 高估值标的风险提示

## 三、市场数据一致性

### 指数数据
- 上证指数、深证成指、创业板指、科创50、北证50
- 涨跌幅与个股涨跌数量一致（涨多跌少 → 指数上涨）

### 市场概览
- 总成交额：各板块成交额之和（合理范围）
- 涨跌家数：涨 3245 / 跌 1678（与指数上涨一致）
- 涨停 68 家 / 跌停 12 家

## 四、订单模型增强

### 新增字段
| 字段 | 类型 | 说明 |
|------|------|------|
| filledQuantity | number | 实际成交量 |
| avgFillPrice | number | 成交均价（VWAP） |
| commission | number | 佣金（0.025%，最低5元） |
| stampTax | number | 印花税（卖出0.05%） |
| rejectReason | string | 拒绝原因 |

### 费用计算公式
```typescript
commission = Math.max(5, totalAmount × 0.00025)
stampTax = side === 'sell' ? totalAmount × 0.0005 : 0
```

### 订单ID格式
```
WT + YYYYMMDD + 6位序列号
例：WT20260618000001
```

### 时间戳
- 生成在 09:30-11:30 或 13:00-15:00 范围内
- 模拟真实交易时段

### 订单状态扩展
```
submitted → matching → partial_filled → filled
                       → cancelled
                       → rejected (附 rejectReason)
```

## 五、K 线趋势一致性

### 改造内容
1. K 线数据趋势与个股涨跌幅方向一致
2. 上涨股票：整体向上趋势，偶有回调
3. 下跌股票：整体向下趋势，偶有反弹
4. 新增 MA5、MA10、MA20 均线指标
5. 均线金叉/死叉信号

---

**改造结果**：所有数据字段之间形成逻辑闭环，用户在任何页面看到的数据都是一致的、可解释的。
