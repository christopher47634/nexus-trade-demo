## P4-A 建议方向：数据层深化

### 方向 1：真实行情接入（推荐）

**目标**: 接入至少一个真实数据源，让首页板块和个股展示实时数据。

**可选方案**:
- 东方财富 API（免费，但不稳定）
- AKShare（Python，数据丰富）
- Tushare（需注册 token）
- 新浪/腾讯行情接口（实时但可能被限流）

**范围**:
- 首页 10 个板块的真实涨跌幅、成交额
- 板块详情页个股列表的真实数据
- K 线图使用真实历史数据

**风险**: API 限流、数据源不稳定、CORS 问题

### 方向 2：交易系统深化

**目标**: 让交易体验更真实。

**可做**:
- 模拟撮合引擎（随机成交价格波动）
- 持仓系统（买入后生成持仓记录）
- 盈亏计算（基于持仓成本和当前价格）
- 交易历史回放

**风险**: 复杂度上升，需平衡 Demo 性质

### 方向 3：社交/分享

**目标**: 增加 Demo 的传播性。

**可做**:
- 交易战绩分享卡片（截图 + 数据）
- Demo 引导完成后生成"体验报告"
- 微信/朋友圈分享优化（OG 图）

### 方向 4：性能与无障碍

**目标**: 提升专业度。

**可做**:
- Lighthouse 分数优化（Performance ≥ 90）
- 完整键盘导航
- ARIA 标签补全
- 深色/浅色主题切换

## 建议优先级

| 优先级 | 方向 | 理由 |
|--------|------|------|
| P1 | 真实行情接入 | 最能提升 Demo 说服力 |
| P2 | 交易系统深化 | 让 Demo 更"真实" |
| P3 | 性能与无障碍 | 专业度加分项 |
| P4 | 社交/分享 | 锦上添花 |

## 技术前提

无论选哪个方向，建议先：
1. 评估当前 mock 数据层的可替换性
2. 确定数据源的稳定性和成本
3. 设计数据层抽象接口，方便后续切换
